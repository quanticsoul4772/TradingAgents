## Rendering rules

- **Currency**: USD; $-prefix; comma thousands; 2 decimal places; never abbreviated (`$103,247.12`, never `$103.2k`).
- **Percent**: signed; 2 decimal places; `%` suffix; never `pp` for symmetry.
- **Date**: ISO 8601 (`2026-05-06`); no localization.
- **Tables**: GFM-style; column-aligned where possible (the renderer should compute column widths).
- **Empty sections**: render the section header + an italic `_No <thing>._` line, NOT an empty table. Exception: skipped entries section is omitted entirely if empty (operator doesn't need a "no skips" notice).
- **Number of recent closes shown**: up to 20 most recent (chronological descending). Older closes remain in state but aren't repeated in every digest.
- **Equity sparkline**: 14 character bars (Unicode `▁▂▃▄▅▆▇█`), one bar per trading day, last 30 days. Plain text alternative below for non-Unicode terminals (the markdown is for files, not terminals — Unicode is fine).

## Disclaimer (R-12, FR-014, SC-005)

The verbatim disclaimer text is the FIRST line of every digest, before the H1 title:

> ⚠️ **Simulation only — not financial advice.** This portfolio is paper-traded for research purposes. No real capital is involved. Past performance of simulated trades does not predict future results. Generated by `tradingagents-lab/scripts/paper_trade.py` (Constitution Principle IV).

The unit test `tests/test_paper_digest.py::test_principle_iv_disclaimer_present` asserts the substring `"Simulation only — not financial advice"` appears in the rendered output.

## File naming

- Pattern: `paper-<portfolio_id>-<YYYY-MM-DD>.md`
- Examples: `paper-default-2026-05-06.md`, `paper-tech-only-2026-05-06.md`
- Default directory: `claudedocs/` (matches the project's existing analysis-doc convention)
- Override via `--digest-dir <path>`
- The harness creates the directory if it doesn't exist (mode 0o755)

---

## Concurrency

- One digest per (portfolio_id, date). Re-running `step` for the same date does NOT overwrite the digest (since `step` exits early per R-5).
- Concurrent reads are safe.
- The harness writes the digest atomically (temp file + rename) to avoid the operator catching a partial file mid-cron.

---

## What the digest does NOT contain

- Per-event JSON dump (see `events_jsonl.md`).
- Stack traces or internal errors (those go to stderr / log).
- LLM model identifiers, prompts, or API costs (none used).
- Forecasts, predictions, or "you should..." language (Principle IV).
- Actual P&L disclaimers about taxes, fees beyond simulated slippage, or real-broker comparisons.
