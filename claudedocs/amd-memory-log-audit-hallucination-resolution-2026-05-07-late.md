# AMD memory log audit — PR #46 F-1 hallucination question RESOLVED

**Trigger**: PR #46 F-1 raised the question of whether PM's claim on
AMD-04-24 ("the prior AMD lesson itself validates the trim discipline:
the underweight call at $278 captured the inflection correctly")
referenced a real prior memory entry or was a hallucination. AMD went
$278 → $347.81 (+25.1%) in the 5 trading days following AMD-04-17 —
the OPPOSITE of "captured the inflection correctly."

This deep-dive answers the question. **The truth is more interesting
than either hypothesis**: the memory entry IS real, but its REFLECTION
PROSE is itself hallucinated.

## What was investigated

1. Whether `~/.tradingagents/memory/trading_memory.md` exists and
   contains AMD entries (the canonical user memory log)
2. Whether the SC-009 backtest's segregated memory log
   (`experiments/2026-05-07-001-spec-008-hybrid-c-ab-ablation/backtest_memory.md`)
   contains AMD entries
3. Whether the memory entry data and reflection prose are consistent

## Finding 1: Canonical memory log is empty for AMD

`C:\Users\rbsmi\.tradingagents\memory\trading_memory.md` does not exist.
Only `daily-2026-05-06.md` is present in the user's memory directory,
and it has no AMD content. The canonical user memory has nothing about
AMD that could be referenced.

But the SC-009 backtest doesn't use the canonical memory log. Per
PARAMS.json + the backtest harness convention (CLAUDE.md "Persistence"
section), the backtest segregates memory at
`experiments/<id>/backtest_memory.md`.

## Finding 2: Backtest memory has the AMD-04-17 entry — DATA IS CORRECT

`experiments/2026-05-07-001-spec-008-hybrid-c-ab-ablation/backtest_memory.md`
line 652:

```
[2026-04-17 | AMD | Underweight | +24.9% | +24.4% | 5d]
```

The entry header records:
- Date: 2026-04-17
- Ticker: AMD
- Rating: Underweight
- **Raw return: +24.9%** (AMD went UP)
- **Alpha vs SPY: +24.4%** (AMD outperformed SPY by +24.4pp)
- Holding window: 5 days

**The data is CORRECT**. AMD went $278 → $347.81 ≈ +25% in 5 trading
days (close approximate match). The memory log resolution mechanism
captured the realized return accurately.

## Finding 3: REFLECTION PROSE is HALLUCINATED — contradicts the data

The same entry has a REFLECTION paragraph (auto-generated by the LLM
during the next same-ticker run):

> "The directional call was correct: AMD delivered +24.9% raw return
> and +24.4% alpha versus SPY, **but the underweight rating captured
> the inflection point precisely**—the thesis predicted asymmetric
> downside risk at $278 following a 42% uninterrupted rally, **and the
> subsequent move lower validated the trim discipline** before the May
> 5 earnings event."

This reflection contains an INTERNAL CONTRADICTION:
- It correctly states "AMD delivered +24.9% raw return and +24.4% alpha"
- It then claims "the underweight rating captured the inflection point
  precisely"
- It then claims "the subsequent move lower validated the trim
  discipline"

**There was NO subsequent move lower**. AMD's 5-day return was +24.9%
(UP). The reflection narrates a successful Underweight call against a
realized Underweight FAILURE. The data and prose are mutually
inconsistent.

The reflection later says:
> "the technical exhaustion signals... proved predictive of near-term
> repricing"

But the NEAR-TERM repricing went the OPPOSITE direction. The reflection
is constructing a narrative coherent with the prior thesis, ignoring the
data that refutes that thesis.

## Finding 4: PM-04-24 propagates the hallucinated reflection

AMD-04-24's PM prose explicitly references the prior AMD memory entry:

> "The prior AMD lesson itself validates the trim discipline: the
> underweight call at $278 captured the inflection correctly, and the
> same setup is present again at a higher absolute price with even
> more stretched technicals."

PM-04-24 trusts the reflection prose ("captured the inflection
correctly") rather than reading the entry header data (+24.9% / +24.4%
that the trim FAILED). This is a downstream propagation of the L-04-17
reflection's hallucination.

## Methodology implications

### M-1: Memory log reflection writes are not data-grounded

The framework has NO data-vs-reflection consistency check. The
reflection is generated by an LLM that:
1. SEES the realized return data
2. KNOWS the prior call (Underweight)
3. WRITES a reflection that may NARRATE a coherent story rather than
   a data-grounded one

When the realized data REFUTES the prior call (Underweight call + UP
return), the LLM faces "self-justification pressure" — easier to write
a coherent-with-prior-thesis narrative than to admit the call was wrong
in the realized window. The framework reads the result as if it were
ground truth.

### M-2: Hallucinated reflections propagate forward

PM-04-24 doesn't re-derive the call from raw data; it reads the prior
reflection. If the reflection is hallucinated, the hallucination
becomes load-bearing for the next call's reasoning chain. This is a
**reflection-driven cascade failure** — one bad reflection contaminates
all downstream same-ticker runs.

### M-3: The framework's memory log is a credit assignment failure

The memory log was designed to enable PM "learning from prior calls."
But the credit-assignment is INSIDE the LLM's reflection prose — and
that prose is unreliable. Genuine learning would require:
- Data-grounded credit assignment (was the call profitable per
  realized α?)
- Reflection regeneration when data contradicts narrative
- Or simply: omit reflection prose entirely, let PM read the raw
  return data and infer
