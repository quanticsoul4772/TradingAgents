# Paper-Trading Harness — Operator Guide

> ⚠️ **Simulation only — not financial advice.** This harness paper-trades for research purposes. No real capital is involved. Past performance of simulated trades does not predict future results. Generated by `tradingagents-lab/scripts/paper_trade.py` (Constitution Principle IV).

A deterministic position-keeping simulator that consumes signals from `scripts/daily_signals.py`, opens/closes virtual long-only positions per a versioned policy, marks to market against yfinance close prices, persists state across days, and emits daily markdown digests.

**Spec**: `specs/002-paper-trading-harness/`. Validated against SC-003 (50-ticker signal-at-scale corpus) — replay reproduces the bullish-bucket mean α within tolerance.

---

## Prerequisites

- `tradingagents-lab` installed editable: `pip install -e .`
- Python 3.10+ (3.12.8 recommended)
- yfinance access (no API key required)
- Optional: existing SC-003 corpus at `experiments/2026-05-05-003-signal-at-scale/results.csv` for the validation-gate test

## Quickstart

### 1. Generate today's signals

```bash
python scripts/daily_signals.py \
    --tickers data/watchlists/tech_weighted.txt \
    --emit-csv ~/.tradingagents/paper/today-signals.csv
```

LLM cost: ~$2-4 for the default 25-ticker watchlist (Opus + Haiku). This is the only LLM-incurring step in the daily flow.

### 2. Update the paper portfolio

```bash
python scripts/paper_trade.py step \
    --signals-csv ~/.tradingagents/paper/today-signals.csv
```

LLM cost: $0 (the harness is a pure signal consumer). Wall-clock: ~10 seconds.

Outputs:
- Updates `~/.tradingagents/paper/default.json`
- Appends events to `~/.tradingagents/paper/default.events.jsonl`
- Writes `claudedocs/paper-default-<today>.md`
- Prints one-line summary to stdout

### 3. Inspect at any time

```bash
python scripts/paper_trade.py status
```

Read-only; renders the current digest to stdout. No state mutation.

## Subcommands

### `replay` — backtest over a date range

```
python scripts/paper_trade.py replay \
    --signals-csv <csv> \
    --watchlist <txt> \
    --start <YYYY-MM-DD> \
    --end <YYYY-MM-DD> \
    [--portfolio-id <str>] \
    [--starting-equity <Decimal>] \
    [--policy <json>] \
    [--digest-dir <path>] \
    [--state-dir <path>] \
    [--no-write-state] \
    [--yes]
```

Deterministic: same inputs always produce the same outputs (subject to yfinance data availability).

### `step` — single trading day, idempotent

```
python scripts/paper_trade.py step \
    --signals-csv <csv> \
    [--date <YYYY-MM-DD>] \
    [--portfolio-id <str>] \
    [--state-dir <path>] \
    [--digest-dir <path>] \
    [--no-write-state]
```

Re-running for the same date is a byte-identical no-op (SC-002). Designed for cron use.

### `status` — current-state inspection

```
python scripts/paper_trade.py status \
    [--portfolio-id <str>] \
    [--state-dir <path>] \
    [--mark-to-date <YYYY-MM-DD>]
```

No state mutation. Renders the digest to stdout using the latest available close prices.

## Default position policy (PolicySnapshot v1-alpha)

| Knob | Default | Notes |
|---|---|---|
| `holding_window_trading_days` | 21 | Matches the empirically-validated horizon |
| `n_max_positions` | 8 | Hard cap on simultaneous open positions |
| `target_per_position_pct` | 10% | Of current equity at moment of entry |
| `cash_buffer_pct` | 10% | Always maintained (of starting_equity) |
| `per_sector_cap_pct` | 50% | Prevents pure-Tech concentration despite SC-003 finding |
| `per_position_cap_pct` | 15% | Hard ceiling on any single position |
| `entry_slippage_bps` | 5 | Buy-side slippage haircut |
| `exit_slippage_bps` | 5 | Sell-side slippage haircut |
| `benchmark` | `SPY` | For α computation |
| `mid_window_exit_on_bear_signal` | `True` | Sell/Underweight closes positions early |
| `re_entry_cooldown_trading_days` | 0 | Same-day re-entry permitted |

Override any default by passing `--policy <json>` with a partial PolicySnapshot file. Policy is locked-in for a portfolio's lifetime — cannot drift mid-portfolio.

**Mid-window Hold rating** is explicitly ignored (Constitution Principle VII guard): held positions ride out the validated 21d window even if a fresh Hold appears on the same ticker. Hold = calibrated abstention applies to fresh entries, not existing positions.

## Persistence

- State JSON: `~/.tradingagents/paper/<portfolio_id>.json` (full materialized portfolio)
- Event log JSONL: `~/.tradingagents/paper/<portfolio_id>.events.jsonl` (append-only audit trail)
- Sector cache: `~/.tradingagents/paper/sectors.json` (shared across portfolios)
- Daily digests: `claudedocs/paper-<portfolio_id>-<YYYY-MM-DD>.md`

Honors `TRADINGAGENTS_CACHE_DIR` env var. Atomic writes via temp-file-rename.

## State file inspection

```bash
jq '.cash, .positions | keys, (.closed | length)' ~/.tradingagents/paper/default.json
```

Filter event log for entries:

```bash
grep '"event_type": "entry"' ~/.tradingagents/paper/default.events.jsonl | jq .payload
```

Replay invariant: applying the JSONL events in order to an empty Portfolio reproduces the JSON state byte-identically (after canonical re-serialization).

## Cron-style daily flow

**Linux/macOS** (weekdays, 17:00 ET):

```cron
0 17 * * 1-5 cd /path/to/tradingagents-lab && \
    .venv/bin/python scripts/daily_signals.py --tickers data/watchlists/tech_weighted.txt --emit-csv ~/.tradingagents/paper/today-signals.csv && \
    .venv/bin/python scripts/paper_trade.py step --signals-csv ~/.tradingagents/paper/today-signals.csv >> ~/.tradingagents/paper/cron.log 2>&1
```

**Windows** (Task Scheduler):

```powershell
$cmd = @'
cd C:\Development\Projects\TradingAgents
.\.venv\Scripts\python.exe scripts\daily_signals.py --tickers data\watchlists\tech_weighted.txt --emit-csv $env:USERPROFILE\.tradingagents\paper\today-signals.csv
.\.venv\Scripts\python.exe scripts\paper_trade.py step --signals-csv $env:USERPROFILE\.tradingagents\paper\today-signals.csv
'@
$action = New-ScheduledTaskAction -Execute "pwsh.exe" -Argument "-Command `"$cmd`""
$trigger = New-ScheduledTaskTrigger -Daily -At "5:00PM"
Register-ScheduledTask -TaskName "tradingagents-paper-daily" -Action $action -Trigger $trigger
```

Idempotency means re-firing on the same day is safe (no double-counting).

## Multiple portfolios (e.g. for policy ablation)

Each `--portfolio-id` gets its own state file + event log. Useful for A/B testing policies.

```bash
# Create alternative-policy override
cat > policy-30d.json <<EOF
{
  "policy_version": "v1-alpha-30d",
  "holding_window_trading_days": 30,
  "target_per_position_pct": "10.0",
  "n_max_positions": 8,
  "cash_buffer_pct": "10.0",
  "per_sector_cap_pct": "50.0",
  "per_position_cap_pct": "15.0",
  "entry_slippage_bps": "5.0",
  "exit_slippage_bps": "5.0",
  "benchmark": "SPY",
  "mid_window_exit_on_bear_signal": true,
  "re_entry_cooldown_trading_days": 0
}
EOF

# Use it
python scripts/paper_trade.py replay \
    --signals-csv ... --watchlist ... \
    --start 2026-01-01 --end 2026-04-30 \
    --portfolio-id policy-30d \
    --policy policy-30d.json --yes
```

Per Constitution Principle II (One Experiment Per Change), this varies one knob at a time vs the default-policy baseline.

## Troubleshooting

### "No portfolio found"
- `portfolio_id` typo, or no `step`/`replay` has ever run for this id
- `TRADINGAGENTS_CACHE_DIR` env var changed since last run

### "Corrupt state file"
- Manual edit broke schema → restore from backup
- Power loss mid-write → check for `.json.tmp` file in same dir

### "Already processed"
- `step` exited early because today is in `equity_curve`. Expected on cron re-fires.
- To force re-run (unsafe; rare): delete that day's `mark` event and remove the `equity_curve` entry. Better: start a new `portfolio_id`.

### "Insufficient cash" on every entry
- Cash buffer floor hit. Lower `cash_buffer_pct` or raise `starting_equity`.

### "Per-sector cap exceeded"
- Signals clustering in one sector. Skip events identify which entries were turned away. Pass a custom policy with `per_sector_cap_pct: 100` for tech-only portfolios.

## What this harness does NOT do

- Execute real trades — broker integration is out of scope (Principle IV)
- Open short positions (FR-007: Sell/Underweight only triggers exits)
- Tax-lot accounting
- Fractional shares (v1 limitation)
- Auto-back-fill missed days (operator-initiated via `replay`)
- Generate signals (use `daily_signals.py` separately)
- Send notifications

## Validation: SC-001 reproduction gate

The integration test `tests/test_paper_sc003_reproduction.py` replays the SC-003 corpus through the harness and asserts the closed-position bullish-bucket mean α reproduces the corpus headline within ±150 bps. Run with:

```bash
pytest tests/test_paper_sc003_reproduction.py -v -m integration
```

Skipped automatically if the SC-003 CSV isn't present.
